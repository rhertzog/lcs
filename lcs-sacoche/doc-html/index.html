Module Lcs-Sacoche

L'installation du module lcs-sacoche installe par dépendance le module lcs-cas.

A la fin de l'installation :

  • l'application SACoche est entièrement installée (aucune autre procédure d'installation n'est nécessaire)
  • l'application SACoche est préconfigurée pour utiliser l'authenfication CAS.

Les comptes utilisateurs "admin" et "webmestre" créés lors de l'installation ne peuvent pas bénéficier de l'authentification CAS.Leurs mots de passe générés aléatoirement lors de l'installation sont envoyés par mail à l'admin LCS.

Se connecter en webmestre :

  • dans administration du site, renseigner les champs d'identifcation de l'établissement
  • dans Informations, changer éventuellement le mot de passe du webmestre

Se connecter en administrateur :

  • créer les comptes utilsateurs Sacoche ( le format du login n'a aucune importance) avec les fichiers Sconet
  • dans Administration générale :
    • -> Importer/Imposer des identifiants
    • -> Importer/Imposer des identifiants LCS
    • -> Dupliquer l'idntifiant LCS

Un compte-rendu des échecs est affiché en fin de procédure. L'association uilisateurSacoche<->utilisateur LCS échoue si un utilisateur Sacoche n'est pas présent dans l'annuaire du LCS, ou s'il a un numéro sconet différent de l'employeeNumber du LDAP ( attention dans l'employeeNumber, le numéro sconet est précédé d'un 0 si c'est un élève et d'un P si c'est un prof )

Dernière version expérimentale :

Version module : 0.1 basé sur la version de SACoche du 20/09/2010.
