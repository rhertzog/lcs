1.       Présentation générale

Nota : dans cette documentation, lorsque nous parlerons d’« installation standard », comprenez qu’il s’agit des paramètres de l’installation de BCDI Web telle qu’elle est livrée, c’est à dire des paramètres que nous avons définis… à ne pas confondre avec les paramètres « par défaut » qui désignent ce qui est prévu (pré-défini) dans BCDI WEB si l’utilisateur ne définit pas lui-même des paramètres.

1.       Objectif

Ce module BCDI web vient compléter le logiciel BCDI réseau client-serveur. Il permet de consulter par interface WEB – Internet ou Intranet - la base de données documentaires gérée par BCDI

2.       Contenu de l’ensemble

BCDI Web livré contient :

Un exécutable (CGI)  BCDIWEB.CGI pour Windows (serveur IIS) ou pour Linux (serveur Apache). Dans cette documentation, le CGI sera appelé « l’application ».

Un modèle de fichier de configuration BcdiWeb_data.ini. Ce fichier devra être adapté à votre propre configuration.

Des fichiers HTM qui serviront de modèles de pages de formulaire et d’édition des résultats. Ces fichiers pourront éventuellement (avec prudence…) être modifiés pour être personnalisés.

3.      Installation

Reportez-vous à la documentation d’installation en ligne.

4.      Le lancement de l’application BcdiWeb

Au lancement de BCDI Web à partir de votre navigateur :

-           vous invoquez directement l’application (lien dans votre page d’accueil)
http://NomDeMonServeur/AliasBcdiWeb/Bcdiweb.cgi/data

vous arrivez directement sur l’écran de recherche correspondant au fichier Index.htm du sous-répertoire Modeles (installation standard \BcdiServ\ProgWeb\modeles, ou /usr/BcdiServ/ProgWeb/Modeles).

2.      Fichier(s) de configuration BcdiWeb_Xxxx.ini (fichier modèle  BcdiWeb_Data.ini)

1.       Structure

A partir du fichier INI modèle (Bcdiweb_Data.ini), vous pouvez créer autant de fichiers Ini que vous avez de bases à mettre en ligne en recopiant puis renommant ce fichier.
A titre d’exemple, BcdiWeb_Data.ini qui vous est proposé, conduit dans la base \BcdiServ\Data, mais vous pouvez créer BcdiWeb_Base1.ini, BcdiWeb_Base2.ini, etc, et ces bases seront alors accessibles depuis un navigateur par les URL
http://MonServeur/MonAlias/BcdiWeb.cgi/Demo
http://MonServeur/MonAlias/BcdiWeb.cgi/Data
http://MonServeur/MonAlias/BcdiWeb.cgi/Base1
http://MonServeur/MonAlias/BcdiWeb.cgi/Base2
etc.

Le fichier INI a la structure classique des fichiers de configuration : il est décomposé en sections, dont le nom est entre crochets, elles-mêmes formées de lignes du type NOM=Valeur.

Les sections peuvent être écrites en minuscules ou majuscules, sans accents. Dans les lignes, les noms peuvent être écrits en minuscules ou majuscules, sans accents. Sous Linux, attention au strict respect de la casse du chemin et du nom des fichiers.

Une ligne commençant par le signe point-virgule (;) est considérée comme un commentaire et n’est donc pas prise en compte.

2.      Section [REPERTOIRES]

1.      DONNEES

La ligne DONNEES indique le répertoire dans lequel doit se trouver la base de données BCDI Abonnement (les fichiers *.DAT et *.IDX). Dans l’installation « standard », le chemin proposé est  c:\bcdiserv\data pour Windows et /usr/Bcdiserv/Data pour Linux.

2.      TRAVAIL

La ligne TRAVAIL indique le répertoire dans lequel l’application pourra écrire ses fichiers temporaires (fichiers *.ECS). Dans l’installation « standard », il s’agira de c:\BcdiServ\ProgWeb ou /usr/BcdiServ/ProgWeb.

3.      MODÈLES

La ligne MODELES indique le répertoire dans lequel doivent se trouver les fichiers modèles HTM de formulaire et de réponses. Dans l’installation « standard », ce répertoire s’appelle Modeles.
Attention ! ce répertoire DOIT être un sous répertoire du répertoire de l’application.

3.      Section [PARAMETRAGES APPLICATION]

1.       PURGE

En minutes, le temps au bout duquel la « mémoire » d’un client est libérée (conservation de la suite de sa recherche, progression de page en page).
(par défaut =10)

2.       RESEAU

Par défaut, le paramètre RESEAU=OUI est pris en compte (même si cette ligne est absente du fichier INI).

La ligne RÉSEAU doit contenir une valeur parmi les deux suivantes :

·         NON (si vous interrogez une copie de la base, il est alors possible de mettre la base à jour par simple copie d’une nouvelle base venant écraser les fichiers de la précédente)

·         OUI, si vous interrogez la base réelle gérée par BCDI réseau en direct.

3.      RESERVATIONS

Laissez RESERVATIONS= NON dans cette version. La prochaine version de BCDI Web permettra les réservations en ligne, il sera alors utile de gérer ce paramètre.

4.      CACHE

Par défaut CACHE=OUI

5.      BLOC

La ligne BLOC doit contenir le nombre maximum de réponses dans une page de liste de résultats. Par défaut, ce nombre est de 10. Si le nombre est 0, il n’y a pas de pagination dans la liste des réponses. Elles sont toutes dans une seule et même page de résultats.

6.      CONNECTIONS

La ligne CONNECTIONS contient le nombre maximum de connexions simultanées gérées par l’application.

7.      REPONSES

La ligne REPONSES, si elle existe, indique le nombre maximum de réponses renvoyées si l’affichage des réponses se fait en une seule page (BLOCS=0) ou Liste Biblio en recherche expert (par défaut, le nombre maximum est fixé à 200).

8.      TIME OUT

La ligne TIME OUT, si elle existe, indique le temps, en minutes, au-delà duquel une session avec un client est interrompue s’il est resté inactif pendant tout ce temps. Par défaut, si cette ligne est absente, ce temps est de 60 minutes.

9.      EMAIL

La ligne EMAIL, si elle existe, indique une adresse de messagerie internet pour contacter une personne responsable du centre documentaire. Dans ce cas, cette adresse s’affichera en tête de la page de liste de réponses.

Pour activer cette fonction d’affichage, vous devrez ajouter à un des fichiers modèles de votre choix (formulaire d’interrogation ou modèle de restitution) la balise <#EMAIL>.

Cette adresse peut être entourée de commentaire. Dans ce cas, la partie MailTo sera la partie contenant le @ et située entre deux blancs.

Exemple : Pour tous renseignements, écrivez à NomDuResponsable@MaMessagerie (le lien ne se fera que sur l’adresse, pas sur tout le texte.

10.   CACHECONNECTIONS

La ligne CACHECONNECTIONS, si elle existe, indique si les connexions sont détruites après utilisations et renouvelées pour chaque nouveau client (NON), ou si elles sont conservées cachées en attendant d’être réutilisées pour un nouveau client (OUI). Par défaut, c’est la deuxième option (OUI) qui fonctionne.

11.    DEBUG

La ligne DEBUG indique éventuellement le nom d’un fichier de log pour le débogage de l’application (après les essais d’installation, il est inutile de conserver ce fichier, placez un point-virgule devant cette ligne).

12.    XML

Par défaut, XML=NON. Autorise ou interdit le retour des résultats au format XML (voir ci-dessous le chapitre concernant XML).

13.    INTERDIT

Permet de paramétrer un complément d’équation de recherche SAUF. Il est ainsi possible de rendre invisible à la recherche des notices, par exemple :
INTERDIT=TEMPORAIRE N.=OUI correspond à l’équation de recherche « SAUF Temporaire N.=~oui~ ».

14.    FILTRE

Permet de paramétrer un complément d’équation de recherche ET. Il est ainsi possible de ne rendre visible qu’une partie des notices, par exemple :
FILTRE=TEMPORAIRE N.=NON correspond à l’équation de recherche « ET Temporaire N.=~non~ », c’est à dire que seules les notices « définitives » d’une base de travail par exemple seront vues.

4.           Section [MODELES]

Elle doit définir le nom des fichiers HTM qui servent de modèles pour l’interface WEB. Il y en a trois, désignés dans les 3 lignes de cette section.

1.       FORMULAIRE

FORMULAIRE : le modèle de page d’accueil pour la recherche. Par défaut, Index.htm (dans l’installation « standard », Index.htm est la page de recherche

2.       LISTE

LISTE : le modèle de page pour la liste des notices résultant de la recherche. Par défaut, Liste.htm (dans la présentation « standard » de BCDI Web, cette page s’appelle ListeNot.htm)

3.      FICHE

FICHE : le modèle de page pour la fiche complète des notices résultant de la recherche. Par défaut, Liste.htm (dans la présentation « standard » de BCDI Web, cette page s’appelle FicheNot.htm)

 

5.          Section [MESSAGES]

1.      PROGRESSION

PROGRESSION = Ecrivez librement le message que vous voulez voir afficher pour commenter la barre de progression en résultats par liste (à adapter au paramètre BLOC, de 10 en 10, de 20 en 20 etc).

2.      LIMITE

LIMITE = Ecrivez librement le message que vos voulez voir afficher pour commenter le fait que toutes les fiches ne seront pas affichées (rappel : fonction activée si l’utilisateur demande un format non progressif) Par défaut, BCDI Web affichera les 200 premières au maximum. Si vous avez décidé d’autoriser plus de réponses en indiquant par exemple REPONSES= 300 dans la section CONNECTIONS, vous pouvez ainsi l’indiquer à l’utilisateur..

6.      .    Section [PAGES ERREURS]

1.       IDENTIFICATION

IDENTIFICATION= nom du fichier HTM qui s’affiche en cas d’erreur de saisie du mot de passe (visite du compte lecteur) 

2.      AUTORISATION

AUTORISATION=nom du fichier HTM qui s’affiche si, bien que bien identifié, l’usager n’est pas autorisé à visiter son compte (dans BCDI,  fiche emprunteur, compte= Non ).

3.      RECHERCHE

RECHERCHE=ErreurRecherche.htm fichier HTM qui s’affiche en cas de recherche sur un Non descripteur en recherche thésaurus.

 

7.       Section [CONSTANTES]

Dans les fichiers Htm modèles, il est possible de créer des balises qui seront automatiquement remplacées par ce que vous aurez défini vous-même.
Par exemple :
REP_IMAGES=../Images/
permettra à BcdiWeb de remplacer à l’affichage la balise <#CONST REP_IMAGES> par  « ../images ».
Vous pouvez ainsi personnaliser automatiquement un fichier modèle en fonction de la base que vous voulez interroger, par exemple, son nom :
NOM_BASE=Base de démonstration (c:\BcdiServ\Demo)
en remplaçant « Base de démonstration (c:\BcdiServ\Demo) » par le nom de votre choix.
Vous pouvez aussi par exemple créer votre propre balise pour changer la couleur, dans le fichier Ini, vous écrivez :
COULEUR_BASE1=EEEEEEE
en vous remplacez, dans les modèles Html
Color=’EEEEEEEE’
 par
Color=’<#CONST COULEUR_BASE1>’

Les balises :
AFFDICO=OUI
AFFDISPONIBILITE=OUI
AFFCOMPTE=OUI
comme leur nom l’indique, permettent de gérer de manière conditionnelle les écrans d’affichage (par exemple, si AFFDISPONIBILITE=oui, l’option Disponibilité est visible)

3.       Balises et conventions dans les modèles

Les modèles proposés peuvent être modifiés (avec prudence !) pour être personnalisés, mais à condition de respecter l’existant (ou des règles strictes) concernant les balises.

Les balises sont placées dans les modèles HTML et sont interprétées par BCDI Web. Elles sont de la forme <#nom-de-la-balise> .

ATTENTION : à moins de modifier ces modèles avec un simple éditeur de texte, il faut vérifier que des éditeurs spécialisés HTML en mode graphique ne vont pas modifier la signalisation de ces balises en remplaçant notamment le crochet ouvrant < par sa valeur interprétée  &lt;  et le crochet fermant > par &gt;

1.      Dans le Formulaire

Les différents formulaires pourront avoir des zones de saisie portant les noms suivants (dans l’installation standard, nous les avons présentés dans des écrans distincts, mais il serait tout à fait possible de regrouper des balises dans un seul et même écran, en respectant bien évidemment une certaine « logique ») :

1.       Ecrans de recherche par Thèmes  (simple et avancé) :

THEMEPOLE1CASE1, THEMEPOLE1CASE2,
THEMEPOLE2CASE1, THEMEPOLE2CASE2,
THEMEPOLE3CASE1, THEMEPOLE3CASE2,
THEMESAUFCASE1, THEMESAUFCASE2
DISPONIBILITE
SUPPORT, et dans la liste Select  <#OPTION_SUPPORT>
il serait possible d’ajouter
DISCIPLINE, et dans la liste Select  <#OPTION_DISCIPLINES>
TYPEDENATURE, et dans la liste Select  <#OPTION_TYPES_NATURE>
NIVEAUX N. , et dans la liste Select  <#OPTION_NIVEAUX>
GENRES, et dans la liste Select  <#OPTION_GENRES>
Nota : il est possible d’indiquer ici une largeur maximale d’affichage (nombre de caractères maximal d’affichage de l’index), syntaxe, par exemple  : <#OPTION_DISCIPLINES largeur=27>, BCDI 3 Web tronquera l’index du champ Disciplines à 27 caractères.
Recherche avec utilisation de Dico :
<input type="hidden" name="LANGAGE" value="DICO">
Recherche sans utilisation de Dico (texte seul):
<input type="hidden" name="LANGAGE" value="TEXTE SEUL">
 Extension de recherche :
<INPUT  type="Checkbox" name="AUTOPOSTAGE">
Classement par pertinence :
<input type="hidden" name="CLASSEMENT" value="PERTINENCE">

2.      Ecran de recherche par critères :

TITRE
TITREMOT
AUTEUR
COLLECTION
EDITEUR

3.      Ecran de recherche Thésaurus

THEMEPOLE1CASE1, THEMEPOLE1CASE2,
THEMEPOLE2CASE1, THEMEPOLE2CASE2,
THEMEPOLE3CASE1, THEMEPOLE3CASE2,

Recherche sur les seuls descripteurs du thésaurus :
name="LANGAGE" value=’THESAURUS’
Extension de recherche :
<INPUT  type="Checkbox" name="AUTOPOSTAGE">

4.      Balises « Constantes »

Voir ci-dessus la section [CONSTANTES] du fichier de paramétrage.

Astuce :
Il serait tout à fait possible, pour personnaliser des écrans d’écrire
<div class="header"><img src="<#CONST REP_IMAGES><#CONST IMAGE_FOND>"
au lieu de
<div class="header"><img src="<#CONST REP_IMAGES>menu-themes.gif"
bien sûr en créant dans la section CONSTANTES de chaque fichier INI une constante :
IMAGE_FOND=Fond1.jpg (dans le premier fichier INI)
IMAGE_FOND=Fond2.jpg (dans le deuxième fichier INI)
etc
de même qu’il serait possible de remplacer une couleur par une constante déclarée…

2.      Dans les résultats : Liste et Fiche 

En installation standard, le fichier ListeNot .htm sert de modèle pour l’affichage des résultats en liste, etr le fichier FicheNot.htm pour l’affichage de la fiche complète.
Ces deux fichiers (ou vos propres fichiers) DOIVENT se trouver dans le sous répertoire MODELES déclaré dans le fichier INI de l’application.
Les balises devront correspondre à la nomenclature de BCDI pour les fichiers modèles HTM.
Voir les pages du site Club-BCDI :

http://club-bcdi.crdp-poitiers.cndp.fr/index.php?rub=nom_balise_modele

4.      .   Recherche directement depuis l’URL

1.       Equation en ligne

On peut lancer une recherche directement dans l’URL de départ. Pour cela, il faut, dans le navigateur internet, donner l’adresse de l’application :
http://MonServeur/MonAlias/bcdiweb.cgi/demo?
suivi d’une requête du type :
?eq:[champ1=valeur1]& [champ1=valeur1]& [champ1=valeur1]…
où champn sont des noms de champs et valeurn des valeurs recherchées.
On peut faire suivre les valeurs d’un tiret pour une extension à droite.

Exemple :
?eq:(auteurs=perr-%20ou%20auteurs=grim-)&editeurs=gal-
devrait donner, entre autres, les livres de Perrault ou de Grimm édités par Gallimard.

 Notez bien que  :
- la requête commence par eq : (eq suivi des deux points) ;
- on peut mettre indifféremment des majuscules ou des minuscules ;
- on peut également ne mettre qu’un mot après le signe eq: Dans ce cas, ce mot sera recherché en plein texte ;
- les espaces doivent être remplacés par  %20 ;
- le nom des champs doit être EXACTEMENT celui qui apparaît dans BCDI ;
- vous pouvez utiliser les opérateurs booléens ET, OU et SAUF (mais en les encadrant de %20) et en pensant bien éventuellement aux nécessaires parenthèses….

Quelques exemples :

- vous pouvez, sur une page Html, créer un lien pour afficher dynamiquement vos nouveautés (ici, les données saisies depuis le premier janvier 2004) :
?eq:date%20de%20saisie>01/01/2007

- ou créer un lien pour afficher dynamiquement une bibliographie thématique :
?eq :(intranet%20OU%20internet)%20ET%20sécurité

Astuce :
Il peut être assez complexe d’écrire à la main une équation de recherche…surtout avec des parenthèses… Il est beaucoup plus simple d’utiliser BCDI en mode expert, de rédiger l’équation de recherche, de la vérifier, puis, par copier-coller de l’ajouter au bout de ?eq : puis de remplacer les espaces par %20.


- vous pouvez passer en paramètres l’utilisation de Dico (« tremblement de terre » permettra de trouver « séisme » entre autres) et le classement par pertinence :
?eq:tremblement%20de%20terre&langage=DICO&classement=PERTINENCE

- vous pouvez demander un classement sur tous les champs indexés de BCDI 3 (sachant que le tri sur les dates est TOUJOURS inverse) : 
?eq:date%20de%20saisie>01/01/2004&classement=date%20de%20saisie

- vous pouvez demander l’édition dans un format normalisé (BIBLIO ou ISBD) :
?eq:date%20de%20saisie>01/01/2004&classement=date%20de%20saisie&format=biblio

2.      Résultats au format  FORMAT XML :

Si, dans le fichier INI, XML=oui, il est possible depuis un navigateur (ou par programmation PHP, ASP ou autre…) de demander à l’application les résultats au format XML :
?eq:date%20de%20saisie>01/01/2004&classement=date%20de%20saisie&format=XML
Par défaut, le format XML renvoyé est le format MémoNotices XML complet de BCDI (voir dans l’aide de BCDI, sommaire, annexe, DTD).
Il est aussi possible de proposer un fichier modèle d’extraction.
?eq:France&format=XML&fichier= FICHE_EXE_XML.TXT
 Pour cela, le fichier correspondant à LISTE devra lui-même être un fichier XML, contenant les balises de votre choix, et, entre ces balises, le nom des champs avec la syntaxe BCDI (cf modèles HTM ci-dessus).
Attention ! Ce fichier DOIT être placé dans le sous-répertoire MODELES.
Dans ce fichier modèle, vous pouvez insérer un autre fichier XML (par exemple dans FICHE_EXE_XML.TXT) qui ‘bouclera’ pour afficher les champs de tous les exemplaires (<#LISTE_EXEMPLAIRES FICHE_EXE_XML.TXT>

Exemple du fichier XML.TXT livré dans MODELES :

<?xml version="1.0" encoding="ISO-8859-1" ?>
<MEMO_NOTICES count="<#nb_reponses>" session="<#connexion>">
<NOTICES>
<TYPE_NOTICE_N><#Champ Notices Type Notice></TYPE_NOTICE_N>
<RESUME_N><#Champ Notices Résumé></RESUME_N>
<DESCRIPTEURS_N><#Champ Notices Descripteurs></DESCRIPTEURS_N>
<MOTS_CLES_N><#Champ Notices Mots clés></MOTS_CLES_N>
<NATURES_N><#Champ Notices Natures></NATURES_N>
<GENRES_N><#Champ Notices Genres></GENRES_N>
<NIVEAUX_N><#Champ Notices Niveaux></NIVEAUX_N>
<PUBLICS_N><#Champ Notices Publics></PUBLICS_N>
<DISCIPLINES_N_N><#Champ Notices Disciplines N.></DISCIPLINES_N_N>
<AUTEURS_N><#Champ Notices Auteurs></AUTEURS_N>
<FONCTIONS_N><#Champ Notices Fonctions></FONCTIONS_N>
<DOCUMENT_N><#Champ Notices Document></DOCUMENT_N>
<COLLATION_N><#Champ Notices Collation></COLLATION_N>
<NOTES_N><#Champ Notices Notes></NOTES_N>
<DOCUMENTS>
<EDITEURS_D><#Champ Documents Editeurs></EDITEURS_D>
<DATE_DE_PARUTION_D><#Champ Documents Date de parution></DATE_DE_PARUTION_D>
<COLLECTION_D><#Champ Documents Collection></COLLECTION_D>
<NO_COLLECTION_D><#Champ Documents N° Collection></NO_COLLECTION_D>
<SUPPORT_D><#Champ Documents Support></SUPPORT_D>
<ISBN_D><#Champ Documents ISBN></ISBN_D>
<ISSN_D><#Champ Documents ISSN></ISSN_D>
<NO_NORMALISE_D><#Champ Documents N° Normalisé></NO_NORMALISE_D>
</DOCUMENTS>

<#LISTE_EXEMPLAIRES FICHE_EXE_XML.TXT>

<RESSOURCE_1_N><#Champ Notices Ressource 1></RESSOURCE_1_N>
<RESSOURCE_2_N><#Champ Notices Ressource 2></RESSOURCE_2_N>
</NOTICES>
</MEMO_NOTICES>
